# BM25 for RAG

## Overview

Section **10** of Phase 7. **BM25** remains essential — dense retrieval misses exact tokens (SKUs, statutes, error codes).

## TF-IDF Intuition

Term frequency (importance in doc) × inverse document frequency (rarity across corpus). BM25 adds length normalization and saturation.

## BM25 Score (intuition)

Rare terms in short relevant docs score highest. Common words ("the", "policy") contribute little.

## Why BM25 Still Matters

| Dense misses | BM25 catches |
|--------------|--------------|
| Exact product ID `SKU-8842` | ✓ |
| Statute `42 U.S.C. § 1983` | ✓ |
| Error `ECONNREFUSED` | ✓ |
| Paraphrased concept | Dense better |

## Hybrid Search

```python
# rank_bm25 + vector → RRF fusion
def rrf(rank_lists: list[list[str]], k: int = 60) -> list[str]:
    scores: dict[str, float] = {}
    for ranks in rank_lists:
        for i, doc_id in enumerate(ranks):
            scores[doc_id] = scores.get(doc_id, 0) + 1 / (k + i + 1)
    return sorted(scores, key=scores.get, reverse=True)
```

## Python Example

```python
from rank_bm25 import BM25Okapi

corpus = [doc.split() for doc in documents]
bm25 = BM25Okapi(corpus)
scores = bm25.get_scores(query.split())
top_indices = sorted(range(len(scores)), key=lambda i: -scores[i])[:10]
```
